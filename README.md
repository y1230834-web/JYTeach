# 复习备考网站（通用 · 多课程）

把每门课的复习资料（一组 `.md`）渲染成网页：

- **多课程**：`courses/` 下每个子文件夹是一门课，左上角可切换，内容整体替换。
- **分级导航**：左侧点开任一文档，在其下展开 `##` / `###` / `####` 各级小标题；点击即滚动跳转到对应区域，并随阅读位置高亮当前小节。
- **讲解型 .md** → 直接排版显示（表格、列表、引用、代码均支持）。
- **做题型 .md** → 判断 / 选择（单选·多选）/ 填空 / 简答·名词解释·论述，前端可交互作答、看正确答案、看解析。
- **错题集** → 任意题卡右上角「☆ 加入错题集」收集；「错题训练」每次进入乱序出题；「错题管理」可逐题删除或清空。数据存浏览器本地，**公网部署后仍可持久化与修改**（详见下文）。

新增一门课：在 `courses/` 下放一个新子文件夹，里面按下文格式放 `.md`，刷新即可，**无需改任何代码**。

## 目录结构

```
JYTeach/                          ← 本项目（网站）
├── courses/                      ← 输入：每个子文件夹 = 一门课
│   ├── 00_示例课程（格式模板）/   ← 写法模板（建议保留作参考）
│   │   ├── 00_使用说明与排版示例.md
│   │   └── 01_全题型交互示例.md
│   ├── 政治学原理/
│   │   └── *.md
│   └── 你的新课程/                ← 以后新增课程放这里
│       └── *.md
├── src/                          ← 网站源码
└── package.json
```

排序：课程与文件名可用数字前缀控制顺序（如 `00_`、`第一章`）；展示时前缀会被自动隐藏。

## 运行 / 构建

需要 [Node.js](https://nodejs.org/)（18+）。

```bash
npm install      # 首次安装依赖
npm run dev      # 本地预览，默认 http://localhost:5173
npm run build    # 生成纯静态站点 dist/（内容已全部内联，可直接托管）
```

dev 模式下新增 / 修改 `courses/` 里的 `.md` 会自动热更新。
内容都在项目内（`courses/`），所以连 Git 仓库做云端自动构建也不会失败。

## 部署到 GitHub Pages

项目已包含 GitHub Actions 配置：`.github/workflows/deploy.yml`。

1. 在 GitHub 新建仓库，把本项目推送到 `main` 或 `master` 分支。
2. 打开仓库的 `Settings` → `Pages`。
3. 在 `Build and deployment` 里，把 `Source` 选为 `GitHub Actions`。
4. 之后每次推送到 `main` / `master`，GitHub 会自动运行 `npm ci`、`npm run build`，并发布 `dist/`。

Vite 已配置 `base: './'`，所以发布到 `https://用户名.github.io/仓库名/` 这种 GitHub Pages 项目地址时，静态资源路径也能正常加载。

## 错题集与持久化（公网部署说明）

错题集保存在**浏览器本地存储 localStorage**（key：`review-wrongset-v1`），所以：

- ✅ 这是**纯前端**实现，部署成静态站点（Netlify / Cloudflare Pages / Vercel 等）后，
  收集 / 删除 / 清空错题都正常工作，并且**刷新、重新部署后依旧保留**。
- ✅ 同一浏览器多个标签页之间会实时同步。
- ⚠️ 数据**按设备 / 按浏览器各自保存，不跨设备同步**——在手机和电脑上各有各的错题集，
  清除浏览器数据也会清空。

> 如果以后需要「跨设备同步 / 多人各自账号」，就必须引入后端（数据库 + 登录）。
> 那是另一套架构，本项目当前刻意保持「零后端、可白嫖静态托管」。

## .md 写作格式（生成新课程时遵循）

网站不依赖固定文件名，而是按 **`##` 小节标题里的关键字** 决定每一节怎么渲染。

### 文件整体

```markdown
# 文档标题                ← 第一个 # 作为页面标题

> 可选引言……             ← # 之后、第一个 ## 之前的内容作为引言

## 小节标题一
…内容…

## 小节标题二
…内容…
```

### 小节类型由标题关键字决定

| 标题包含关键字 | 渲染为 |
|---|---|
| `判断` | 判断题交互 |
| `选择` / `单选` / `多选` / `不定项` | 选择题交互（按答案字母数自动判定单选 / 多选） |
| `填空` | 填空题交互 |
| `名词解释` / `简答` / `论述` / `问答` / `思考题` | 主观题交互（作答框 + 参考答案） |
| 其它（考点概览、速记、备考指南…） | 普通 markdown 直接显示 |

> 兜底：命中关键字但解析不出任何题目（例如其实是张表格）会自动按普通 markdown 显示，不会出错。

### 1. 判断题

```markdown
## 二、判断题

1. 题干文字。（**√**）
2. 另一道题，答案为错。（**×**）
   - 解析：可选，写错在哪、正确表述是什么。
```

- 答案写在题干**末尾**括号里并加粗：`（**√**）` 或 `（**×**）`（也支持 `对` / `错`）。
- 解析可选，紧跟一行 `- 解析：……`。

### 2. 选择题（单选 / 多选）

```markdown
## 三、不定项选择题

12. 题干……（**ABD**）
    A 选项一  B 选项二  C 选项三  D 选项四
    - 解析：可选。
```

- 第一行：`编号. 题干……（**正确答案字母**）`。括号里**一个字母 = 单选**，**多个字母 = 多选**。
- 选项行：可写在**一行**（`A x B y C z D w`），也可**每行一个**（`A. 选项一`）。可多于 4 项。
- 多选判分：多选、少选都算错（与考试一致）。
- ⚠️ 一行多选项时，选项正文里不要出现**孤立的大写字母 A–Z**（会被当成分隔符）；中文不受影响。

### 3. 填空题

```markdown
## 五、填空题

1. 新中国成立于 ____ 年，首都是 ____。（**1949｜北京**）
   - 解析：可选。
```

- 题干用 `____`（≥2 个下划线）表示空位。
- 答案写在末尾括号里，**多个空用 `｜` 分隔**（也支持 `|` / `；`）。
- 作答按去空格、忽略大小写比对；提交后逐空标记对错并给出正确答案。

### 4. 名词解释 / 简答 / 论述

```markdown
## 六、简答题

**5. 政府具有哪些特征？**
- ① 阶级性：……
- ② 公共性：……

**6. 名词也用同样写法：**
参考答案支持段落、列表、**加粗**、> 引用 等 markdown。
```

- 每题以**整行加粗的标题**开头：`**编号. 问题？**`。
- 从该行**之后**到下一个 `**…` 行，都是这道题的参考答案。
- 页面为每题提供作答框，点「显示参考答案」展开。

## 想新增一种题型？（开发者）

架构按题型注册表设计，扩展点集中在两个文件：

1. `src/registry.js`：在 `QUESTION_TYPES` 里加一项 `{ type, match, parse }`
   （`match(heading)` 判定标题、`parse(lines)` 解析出题目数组）。
2. `src/components.jsx`：在 `RENDERERS` 里加一个同名 `type` 的 React 组件。

其余（文档解析、课程切换、目录导航）无需改动。

## 技术栈

[Vite](https://vitejs.dev/) + [React](https://react.dev/) + [marked](https://marked.js.org/)。
课程加载见 `src/content.js`（`import.meta.glob('../courses/*/*.md')`）；
文档解析见 `src/parser.js`；题型规则见 `src/registry.js`；组件见 `src/components.jsx`。
